    Site APIs

    reference
    Use the Site API to manage sites, including the ability to create, update, publish, disable, restore, and delete sites and to generate login links and set publish credentials.

    Sites are associated with a particular user and plans. Plans determine the level of functionality available to a site. For example, higher-level plans allow eCommerce functionality. Every site has a unique domain. Each also has login credentials for users that have access to the site to edit and manage it.

    Sites can be published, and you can also suspend and restore sites, as well as disable and enable them.

    GET
    Retrieve a List of Sites

    Returns all sites for the given user. Only list fields are returned.

    Query Parameters

    Returned values are paginated. You can further filter results using these parameters:

    Parameter Description Type
    page Which page of results to return. Start is 1.
    Note: Only the first 25 results are returned by default. If the result set is expected to be over 25, you must implement pagination.
    		integer
    limit Number of results per page to return. Default is 25.
    Max is 200.    		 integer
    filterby Field name to set a filter on. Valid value is role. string
    filterfor ​Value to search the filterby field for.
    Valid values are:
    • owner: Return only sites that the user owns.
    • contributor: Return only sites that the user is a contributor to
    • null: Return all sites the user owns and is a contributor to.
